2. 강의 — claude.ai 웹, 어디에 뭐가 있나?

처음 열면 메뉴가 많아 당황할 수 있습니다. 수업에서 쓸 메뉴 4가지만 파악해 두면 충분합니다.

Projects (프로젝트) 좌측 사이드바

업무 유형별로 대화를 묶어 두는 폴더입니다. 프로젝트마다 "지침(Instructions)"을 저장할 수 있어, 매번 역할을 설명하지 않아도 됩니다.

교사 예시 [담임업무] 프로젝트 지침에 "나는 고2 3반 담임이고, 학생은 30명입니다."를 저장해 두면 이후 모든 대화에 자동 적용됩니다.
Artifacts (아티팩트) 대화창 오른쪽 패널

클로드가 만든 결과물(표·웹페이지·코드 등)이 오른쪽 패널에 따로 펼쳐지는 기능입니다. 텍스트와 결과물을 분리해서 볼 수 있습니다.

교사 예시 "시험 일정 안내 가정통신문을 표로 만들어 줘"라고 하면 오른쪽 패널에 완성된 표가 펼쳐집니다. 바로 복사해서 한글이나 워드에 붙여넣기 가능합니다.
Code Execution (코드 실행) 웹·데스크탑 공통

클로드가 직접 계산·분석·파일 변환을 실행하는 기능입니다. 단순한 텍스트 생성이 아니라 실제 연산 결과를 보여줍니다. PDF 파일도 만들 수 있습니다.

교사 예시 "이 성적 데이터를 평균·표준편차·등수까지 계산해서 표로 보여줘"라고 하면 클로드가 직접 계산한 결과를 출력합니다.
Memory (메모리) 유료 플랜

대화가 끝난 후에도 클로드가 중요한 내용을 기억합니다. 다음 대화를 새로 시작해도 이전에 알려준 정보를 유지합니다.

교사 예시 한 번 "나는 수학 교사이고 고3 수업을 담당합니다"라고 말해 두면, 다음 번 새 대화에서도 클로드가 그 맥락을 기억합니다.
💡 오늘 수업에서 핵심은 Projects입니다. Artifacts와 Code Execution은 쓰면 자동으로 나타나니 별도 설정이 필요하지 않습니다.

4. 강의 — 핵심 기능 3가지

AI

모델 선택 & 응답 품질 조절

Claude Sonnet은 속도와 품질의 균형이 뛰어나 교원 업무에 적합합니다. 긴 공문이나 상세한 분석이 필요할 때는 Claude Opus를 활용하면 더 깊은 분석이 가능합니다.

모델을 바꿔도 이전 대화 맥락은 유지됩니다.
Re

답변 재생성 & 이어쓰기

마음에 들지 않는 답변은 재생성 버튼으로 즉시 재생성할 수 있습니다. '계속 작성해 줘.'를 입력하면 내용을 이어서 생성하며, '3번 항목만 더 구체적으로 수정해 줘.'처럼 부분 수정도 가능합니다.

'마음에 안 들면 재생성' 패턴을 습관화하면 완성도가 높아집니다.
UP

파일 업로드 & 문서 분석

PDF, 이미지, Word 파일을 첨부하면 클로드가 내용을 읽고 분석합니다. 학교 규정집 PDF를 업로드하고 '3장 내용을 요약해 줘.'처럼 활용할 수 있습니다.

한 번에 최대 5개 파일 첨부가 가능합니다.

5. 강의 — 프롬프트 작성법이 중요한 이유

같은 요청도 작성 방식에 따라 결과물의 완성도가 크게 달라집니다.

X Before — 막연한 프롬프트
"사업 계획서 써줘."
  • 어떤 사업인지 몰라서 일반적인 내용만 생성됩니다.
  • 학교·대상·예산 등 맥락이 없어 실제로 쓸 수 없는 내용이 나옵니다.
  • 형식이 지정되지 않아 표·목차 없이 줄글로 출력됩니다.
  • 분량 제한이 없어 너무 길거나 너무 짧게 나옵니다.
결과: 다시 처음부터 설명해야 하는 '반복 수정 지옥'이 시작됩니다.
O After — 4요소 프롬프트
역할 당신은 10년 차 기획부장 교사입니다.
맥락 720명 규모 특성화고, 진로 탐방 사업 기획 중
형식 A4 1장 기안문 형식, 목적·일정·예산 포함
제약 예산 200만 원 이내, 800자 이내
  • 역할이 명확해 전문적인 시각으로 내용이 생성됩니다.
  • 학교 맥락이 담겨 바로 사용 가능한 결과물이 나옵니다.
  • 형식 지정으로 표·항목이 정확히 포함됩니다.
  • 분량 제한으로 한 번에 적절한 길이가 완성됩니다.
결과: 첫 시도에 80% 완성도 달성, 수정은 1~2회로 마무리됩니다.

6. 강의 — GEAR 질문법

좋은 프롬프트의 4가지 재료(역할·맥락·형식·제약)를 단계별로 조립하는 실전 프레임워크입니다.

G

Goal

달성할 목표 + 역할을 제시합니다.

역할(Role)
예시 "당신은 10년 차 고등학교 기획부장 교사입니다. 진로 탐방 사업 계획서 초안을 작성해 줘."
E

Explain

학교·업무 배경을 설명합니다.

맥락(Context)
예시 "우리 학교는 학생 720명 규모 특성화고이며, 내년도 사업 기획 중입니다."
A

Ask

형식과 제약 조건을 요청합니다.

형식(Format) 제약(Constraints)
예시 "A4 1장 기안문, 목적·일정·예산 포함, 예산 200만 원 이내, 800자 이내"
R

Review

결과를 검토하고 수정을 지시합니다.

GEAR의 핵심 단계
예시 "3번 항목을 더 구체적으로 써 줘."
"예산 표를 추가해 줘."
G·E·A 세 단계가 역할(Role)·맥락(Context)·형식(Format)·제약(Constraints) 4가지 재료를 자연스럽게 포함합니다. R(Review) 단계가 GEAR의 핵심 — 첫 답변에 만족하지 말고 반드시 이어서 수정 지시를 해보세요.

강사 시연

7. 강사 시연 — GEAR로 완성한 프롬프트

G·E·A 세 단계를 한 프롬프트에 조립한 뒤 R(Review)로 수정하는 전 과정을 실제로 보여드립니다.

완성된 GEAR 프롬프트

[G — 역할 + 목표]
당신은 10년 차 고등학교 기획부장 교사입니다.
진로 탐방 사업 계획서 초안을 작성해 줘.

[E — 맥락]
우리 학교는 학생 720명 규모의 특성화고이며,
내년도 학생 진로 탐방 사업을 기획 중입니다.

[A — 형식 + 제약]
사업 목적·대상·일정·예산 항목을 포함한
A4 1장 기안문으로 작성해 줘.
예산 200만 원 이내, 분량 800자 이내.

R — 1차 수정 지시

"3번 항목(추진 일정)을 더 구체적으로, 월별 일정표로 써 줘."
첫 답변이 80%라면 R 한 번으로 95%가 됩니다.

R — 2차 수정 지시

"예산 항목을 표 형식으로 바꿔 줘."
형식만 바꿔도 문서 완성도가 달라집니다.
💡 핵심 습관 — 첫 답변을 받은 후 반드시 한 번 이상 R(Review)을 실행하세요. R을 생략하면 GEAR의 절반만 쓰는 것입니다. 이제 직접 해봅니다!

선생님 실습

8. 선생님 실습 — 학부모 안내 문자 작성

협조 요청, 긴급 공지, 일정 안내 상황에 맞는 100자 이내 문자 메시지를 빠르게 완성합니다. (claude.ai 웹 사용)

✏️ 복사는 하되 [ ] 대괄호(상황 유형·내용·대상)는 직접 바꿔 전송하세요. 대괄호만 교체하면 모든 안내 문자에 재사용됩니다.
01
가정통신 10분

학부모 안내 문자 — 협조 요청·긴급 공지·일정 안내

상황 유형과 내용만 입력하면 날짜·조치 사항·문의처가 포함된 간결한 안내 문자가 완성됩니다.

다음 상황에 맞는 학부모 안내 문자를 작성해 줘.

상황 유형: [협조 요청 / 긴급 공지 / 일정 안내 중 선택]
상황 내용: [예: 내일 방송부 학생 오전 8:30 조기 등교
           / 태풍으로 내일 임시 휴업
           / 급식 메뉴 변경 안내]
수신 대상: [해당 학생 학부모 / 전교생 학부모]

조건: 100자 이내, 날짜·조치 사항·문의처 포함, 공식적이고 간결하게 작성해 줘.

결과 예시 — 유형별 완성 문자

협조 요청 예시 (94자)

[방송부] 내일(6/14) 행사 준비로 방송부 학생은 오전 8:30까지 등교해 주시기 바랍니다. 문의: 담임 010-0000-0000

긴급 공지 예시 (88자)

[긴급] 태풍 영향으로 내일(6/15) 전교생 임시 휴업을 결정하였습니다. 가정 지도 부탁드립니다. 교무실 000-0000

[상황 유형]과 [상황 내용]만 바꾸면 어떤 안내 문자 상황에도 바로 적용할 수 있습니다. 결과물이 100자를 넘으면 '100자 이내로 줄여 줘.'라고 이어서 지시합니다.
추가 도전 과제 상황 유형을 '일정 안내'로 바꿔 학교 행사 안내 문자를 직접 작성해 봅니다. 또는 다국어 번역을 요청해 봅니다. 예) "위 문자를 영어로도 번역해 줘."

선생님 실습

10. 선생님 실습 — 실전 프롬프트 실습 3가지

각 프롬프트를 복사해 클로드에 바로 붙여넣어 결과물을 확인합니다.

01
세특 기재 20분

교과 세특(세부 능력 및 특기 사항) 기재 문구 자동 생성

학생별 활동 키워드만 바꾸면 반복 사용이 가능합니다. 30명 세특을 30분 내 완성할 수 있습니다.

당신은 화학 교사입니다.

다음 학생의 활동 내역을 바탕으로 교과 세특 문구를 작성해 줘.
- 교과: 화학 I  학년: 1학년
- 단원: 이온결합
- 성취기준: [해당 성취기준 코드]
- 학생 활동 키워드: 실험 설계, 팀 토론, 보고서 작성
- 특이 사항: 실험 설계에서 창의적인 방법을 제안하고 팀을 이끔

조건: 200자 이내 / 수동태 금지 / 구체적 행동 서술 / 교과 역량 중심으로 작성해 줘.
[교과], [단원], [활동 키워드]만 바꾸면 모든 교과의 세특 문구에 바로 활용할 수 있습니다.
02
평가 기준안 8분

루블릭(평가 기준안) 제작

교과별 성취기준이 자동으로 반영되어 평가 기준의 일관성이 확보됩니다.

고1 통합사회 프로젝트 수행평가 루블릭을 다음 조건으로 만들어 줘.

- 평가 과제: 지역사회 문제 탐구 보고서 작성
- 평가 요소: 내용 이해 / 자료 활용 / 표현력
- 성취 수준: 매우잘함 / 잘함 / 보통 / 노력요함 (4단계)

각 수준별 기술어를 명확하고 구체적으로, 교사와 학생이 모두 이해할 수 있도록 표 형식으로 작성해 줘.
[교과], [평가 과제], [평가 요소]만 바꾸면 모든 수행평가에 재사용할 수 있습니다.
03
수준별 활동지 7분

수준별 맞춤형 활동지 제작

한 번 요청으로 3가지 버전이 동시에 생성됩니다. 학습 격차 해소 활동지에 활용하세요.

영어 관계대명사 단원을 대상으로 수준별 활동지를 만들어 줘.

학습 목표: 관계대명사 who, which, that의 쓰임을 이해하고 문장에서 활용할 수 있다.

다음 3가지 수준으로 각각 A4 1장씩 만들어 줘.
- 상: 심화 탐구 + 서술형 문항 포함
- 중: 기본 개념 확인 + 단답형
- 하: 핵심 내용 채우기 + 예시 중심

각 활동지에는 읽기 자료와 형성평가 문제를 각각 포함해 줘.
[교과], [단원], [학습 목표]만 바꾸면 모든 교과·단원에 바로 적용할 수 있습니다.
심
11. 심화 활용 — 반 단위 행동특성 일괄 생성
엑셀에 정리한 특성 표 → 반 전체 행동특성을 한 번에 (실습용 샘플: 행동특성_실습_샘플.xlsx)
평가·기록 · 심화
반 전체 행동특성 및 종합의견 한 번에 작성

엑셀에서 번호 + 특성 칸만 복사해(이름 제외) claude.ai에 붙여넣고 아래 프롬프트를 함께 보냅니다. 결과를 "번호 | 행동특성" 표로 받아 엑셀에 붙여넣습니다. 학생이 많으면 10명씩 끊어서 진행합니다.

↓ 실습용 샘플 엑셀 내려받기 (행동특성_실습_샘플.xlsx) 
아래 데이터는 탭으로 구분된 학생별 정보입니다. 번호가 바뀌면 새 학생입니다.
다음은 우리 반 학생들의 활동 키워드입니다. 각 학생의 행동특성 및 종합의견을 작성해 줘.

[형식]
- 학생마다 3~4문장, 300~500자, 모든 문장 '~함' 종결, 공식적·객관적 어투
- 학습태도 → 교우관계 → 특기 순서
[조건]
- 구체적 관찰 사례 포함, 부정적 내용은 발전 가능성과 함께 서술
- 학생마다 표현과 어휘를 다르게 하여 문구가 겹치지 않게
- 결과를 "번호 | 행동특성 및 종합의견" 표로 정리해 줘

[학생 데이터]
1 | 수업 집중도 높고 과제 성실 | 친구들과 두루 원만 | 수학 경시 입상
2 | 끈기 있게 끝까지 | 협동 활동 적극 | 농구 등 체육 특기
(번호·특성 칸을 끝까지 붙여넣기)
생성 결과 예시
1 | 평소 수업에 성실히 참여하며 맡은 과제를 끝까지 책임지는 태도가 돋보임. 수학 교과에서 우수한 문제 해결력을 보이며 교내 경시대회에서 성과를 거둠. 친구들과 두루 원만하게 지내며 또래를 돕는 배려심이 있음.
2 | 어떤 활동이든 끈기 있게 끝까지 해결하려는 자세가 인상적임. 모둠 활동에서 협동적으로 참여하며 분위기를 이끎. 농구 등 체육 활동에 특기를 보여 체육대회에서 반을 대표함.
POINT
이름 대신 번호만 입력(개인정보) · 10명씩 분할 · 생기부 기재 전 교사 검토 필수. 교육부도 생성형 AI를 '윤문 등 보조 수단'으로 허용하되 최종 책임은 교사에게 있습니다(소속 교육청 지침 확인).

12. ★ 보너스 실습 — 카카오톡 나와의 채팅 연결해 보기

시간 여유가 있을 때 진행합니다. Claude가 만든 내용을 카카오톡으로 바로 받을 수 있습니다.

📋 연결 방법 (최초 1회)

1
Claude.ai → 프로필 → 설정 → 커넥터 → [커넥터 둘러보기] → "PlayMCP" 검색 → [연결] → 카카오 계정 로그인 → OAuth 동의
2
playmcp.kakao.com/toolbox 접속 → + MCP 추가하기 → "카카오톡 나챗방" 검색 → 추가
3
Claude.ai로 돌아와 커넥터 설정에서 PlayMCP 연결 해제 후 재연결 (도구 목록 갱신)
4
채팅창 좌하단 + 버튼 → 커넥터 → PlayMCP 활성화 → 카카오톡 나와의 채팅 전송 가능!

💬 교사 활용 예시 3가지

작업 완료 후
"오늘 만든 세특 문구 핵심만 요약해서 카카오톡 나와의 채팅에 보내줘."
이동 중 참고
"지금 만든 가정통신문 내용 카카오톡 나와의 채팅으로 보내줘."
할 일 정리
"내일 수업 준비 체크리스트 3가지 만들어서 나와의 채팅에 보내줘."
텍스트만 전송됩니다. 표·서식이 있는 결과물은 "요약해서" 또는 "텍스트로 정리해서"라고 요청하세요.
